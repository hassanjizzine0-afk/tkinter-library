
### Достоинства EKF

- Первое практическое решение проблемы SLAM
- Доказательство сходимости для линейного гауссовского случая
- Эффективен для среднеразмерных сцен

### Недостатки EKF

- Может расходиться при больших нелинейностях
- Работает только с одномодальными распределениями
- Квадратичная сложность $O(n^2)$ по числу ориентиров $n$
- Требует вычисления якобианов

---

## 4. Применение EKF в SLAM (EKF SLAM)

EKF SLAM является одним из первых и наиболее известных решений задачи одновременной локализации и построения карты [Course 05].

### Постановка задачи SLAM

**Дано:**
- Управляющие воздействия $\mathbf{u}_{1:T} = \{\mathbf{u}_1, \mathbf{u}_2, \ldots, \mathbf{u}_T\}$
- Наблюдения $\mathbf{z}_{1:T} = \{\mathbf{z}_1, \mathbf{z}_2, \ldots, \mathbf{z}_T\}$

**Требуется найти:**
- Карту среды $m$
- Траекторию робота $\mathbf{x}_{0:T} = \{\mathbf{x}_0, \mathbf{x}_1, \ldots, \mathbf{x}_T\}$

### Представление состояния в EKF SLAM

Состояние системы включает положение робота и координаты всех ориентиров:

$$
\bm{\mu}_t = \begin{pmatrix}
\mu_{t,x} \\
\mu_{t,y} \\
\mu_{t,\theta} \\
\mu_{1,x} \\
\mu_{1,y} \\
\vdots \\
\mu_{n,x} \\
\mu_{n,y}
\end{pmatrix}, \qquad
\bm{\Sigma}_t = \begin{pmatrix}
\Sigma_{rr} & \Sigma_{r1} & \cdots & \Sigma_{rn} \\
\Sigma_{1r} & \Sigma_{11} & \cdots & \Sigma_{1n} \\
\vdots & \vdots & \ddots & \vdots \\
\Sigma_{nr} & \Sigma_{n1} & \cdots & \Sigma_{nn}
\end{pmatrix}
$$

Размерность пространства состояний составляет $2N + 3$, где $N$ — количество ориентиров.

### Модель наблюдения дальность-угол (Range-Bearing)

$$
\hat{\mathbf{z}}_t^i = \begin{pmatrix}
\sqrt{q} \\
\text{atan2}(\delta_y, \delta_x) - \mu_{t,\theta}
\end{pmatrix}
$$

где $\delta_x = \mu_{j,x} - \mu_{t,x}$, $\delta_y = \mu_{j,y} - \mu_{t,y}$, $q = \delta_x^2 + \delta_y^2$.

### Замыкание цикла (Loop Closing)

Важнейшей особенностью EKF SLAM является возможность замыкания цикла — распознавания уже посещённых участков карты. При успешном замыкании цикла неопределённость в оценках положения робота и ориентиров существенно уменьшается.

> **Важно:** Ошибочное замыкание цикла может привести к расходимости фильтра.

### Сложность EKF SLAM

| Характеристика | Значение |
|----------------|----------|
| Временная сложность | $O(n^2)$ на шаг |
| Память | $O(n^2)$ |
| Основной фактор | Количество ориентиров $n$ |

## 5. Сигма-точечный фильтр Калмана (Unscented Kalman Filter – UKF)

Альтернативой линеаризации через разложение Тейлора является сигма-точечное преобразование (Unscented Transform), используемое в UKF [Course 06].

### Принцип сигма-точечного преобразования

1. Выбирается набор сигма-точек ($\bm{\chi}^{[i]}$) с весами ($w^{[i]}$)
2. Каждая точка преобразуется через нелинейную функцию
3. Из преобразованных точек восстанавливается гауссовское распределение

### Выбор сигма-точек

$$
\mathbf{X}^{[0]} = \bm{\mu}
$$

$$
\mathbf{X}^{[i]} = \bm{\mu} + \left( \sqrt{(n + \lambda) \bm{\Sigma}} \right)_i \quad \text{для } i = 1, \ldots, n
$$

$$
\mathbf{X}^{[i]} = \bm{\mu} - \left( \sqrt{(n + \lambda) \bm{\Sigma}} \right)_{i-n} \quad \text{для } i = n+1, \ldots, 2n
$$

### Веса сигма-точек

$$
w_m^{[0]} = \frac{\lambda}{n + \lambda}
$$

$$
w_c^{[0]} = w_m^{[0]} + (1 - \alpha^2 + \beta)
$$

$$
w_m^{[i]} = w_c^{[i]} = \frac{1}{2(n + \lambda)} \quad \text{для } i = 1, \ldots, 2n
$$

### Преимущества UKF перед EKF

| Характеристика | EKF | UKF |
|----------------|-----|-----|
| Точность аппроксимации | 1-й порядок | 2-й порядок и выше |
| Необходимость вычисления якобианов | Да | Нет |
| Работа с сильной нелинейностью | Плохо | Хорошо |
| Вычислительная сложность | Ниже | Незначительно выше |

## 6. Информационный фильтр (Extended Information Filter – EIF)

Наряду с моментальным представлением гауссовских распределений (математическое ожидание $\bm{\mu}$ и ковариация $\bm{\Sigma}$) существует каноническое представление через информационную матрицу $\bm{\Omega}$ и информационный вектор $\bm{\xi}$ [Course 07].

### Каноническое представление

$$
\bm{\Omega} = \bm{\Sigma}^{-1}, \qquad \bm{\xi} = \bm{\Sigma}^{-1} \bm{\mu}
$$

### Сравнение сложности операций

| Операция | Моментальное представление (KF) | Каноническое представление (IF) |
|----------|--------------------------------|--------------------------------|
| Предсказание (Prediction) | Быстрое | Медленное |
| Коррекция (Correction) | Медленное | Быстрое |

### Вывод

EIF — это EKF в информационной форме. Выбор между KF и IF определяется конкретным применением:
- KF: эффективное предсказание, медленная коррекция
- IF: медленное предсказание, эффективная коррекция

На практике EKF более популярен, чем EIF.

---

## 7. Сравнительный анализ методов фильтрации

| Метод | Линеаризация | Вычисление якобианов | Точность | Сложность |
|-------|--------------|---------------------|----------|-----------|
| KF | Не требуется (линейный) | Нет | Оптимальная (для линейных) | $O(k^{2.4})$ |
| EKF | Тейлор 1-го порядка | Да | Хорошая (умеренные нелинейности) | $O(n^2)$ |
| UKF | Сигма-точки | Нет | Лучшая (сильные нелинейности) | Незначительно выше EKF |
| EIF | Тейлор 1-го порядка | Да | Как у EKF | $O(n^2)$ |

## 🗺️ Система координат и TF (Transform Library)

### Что такое фреймы?

**Фрейм** — это система координат. Каждый сенсор и часть робота имеют свой собственный фрейм.

![image_alt](https://github.com/hassanjizzine0-afk/hector_slam/blob/main/TF.png?raw=true)

### Наша настройка фреймов

| Фрейм | Назначение |
|-------|------------|
| `map` | Глобальная неподвижная система отсчёта (мир) |
| `laser` | Система координат LiDAR-сенсора |

### Цепочка фреймов

```
map ──────→ laser
  │            │
  │            └─ Положение LiDAR-сенсора
  └─ Глобальная система отсчёта (неподвижна)
```

### Зачем нужно статическое преобразование?

```bash
rosrun tf static_transform_publisher 0 0 0 0 0 0 map laser 100
```

**Это сообщает ROS:** Фрейм `laser` находится точно в том же положении и ориентации, что и фрейм `map`.

**Что означают 6 чисел:**

```
Положение: 0 0 0    → смещение по X, Y, Z (метры)
Поворот:   0 0 0    → рысканье, тангаж, крен (радианы)
```

## ⚙️ Параметры

### Параметры фреймов

| Параметр | Значение по умолчанию | Наше значение | Причина |
|----------|----------------------|---------------|---------|
| `_map_frame` | `map_link` | `map` | Глобальный фрейм карты |
| `_base_frame` | `base_link` | `laser` | Лидар — центр робота |
| `_odom_frame` | `odom` | `laser` | Нет колёсной одометрии |

### Параметры построения карты

| Параметр | Значение по умолчанию | Наше значение | Причина |
|----------|----------------------|---------------|---------|
| `_map_resolution` | `0.025` | `0.05` | 5 см на пиксель — хороший баланс |
| `_map_size` | `1024` | `2048` | Карта 102×102 метра |

### Пороги обновления (КРИТИЧЕСКИ ВАЖНЫ!)

| Параметр | Значение по умолчанию | Наше значение | Зачем менять? |
|----------|----------------------|---------------|---------------|
| `_map_update_distance_thresh` | `0.4 м` | `0.05 м` | Обновлять после смещения всего на 5 см |
| `_map_update_angle_thresh` | `0.9 рад` (51°) | `0.05 рад` (3°) | Обновлять после малых поворотов |

> **⚠️ Вот почему ваша карта не обновлялась!** Стандартные пороги слишком велики для малых перемещений.

### Параметры фильтрации лазера

| Параметр | Значение по умолчанию | Наше значение | Причина |
|----------|----------------------|---------------|---------|
| `_laser_min_dist` | `0.4 м` | `0.4 м` | Игнорировать объекты ближе 40 см |
| `_laser_max_dist` | `30.0 м` | `5.0 м` | Максимальная дальность RPLIDAR A1 — 6 м |
| `_scan_topic` | `/scan` | `/scan` | Стандартное имя топика лазера |

## 🚀 Запуск системы (5 терминалов)

```bash
sudo chmod 666 /dev/ttyUSB0
```

```bash
roscore
```

> **Держите это запущенным!** Это мастер ROS.

### Терминал 2: Драйвер RPLIDAR

```bash
rosrun rplidar_ros rplidarNode \
    _serial_port:=/dev/ttyUSB0 \
    _serial_baudrate:=115200 \
    _frame_id:=laser \
    _inverted:=false \
    _angle_compensate:=true
```

> **Держите это запущенным!** Публикует топик `/scan` с данными LiDAR.

### Сводка параметров драйвера

| Параметр | Что делает | Ваша настройка |
|----------|------------|----------------|
| `frame_id` | Имя системы координат лидара | `laser` |
| `inverted` | Нормальное или обратное сканирование | `false` (нормальное) |
| `angle_compensate` | Заполнение пропусков в данных | `true` (сглаживание) |

### Терминал 3: Статическое преобразование (связывает фреймы)

```bash
rosrun hector_mapping hector_mapping \

    _map_resolution:=0.05 \
    _map_size:=2048 \
    _map_update_distance_thresh:=0.05 \
    _map_update_angle_thresh:=0.05 \
    _laser_max_dist:=5.0
```

> **Держите это запущенным!** Строит карту из данных LiDAR.

## ❓ Устранение проблем

| Проблема | Решение |
|----------|---------|
| Permission denied на `/dev/ttyUSB0` | `sudo chmod 666 /dev/ttyUSB0` |
| Карта не обновляется | Уменьшить `map_update_distance_thresh` и `map_update_angle_thresh` |
| Нет лазерных лучей в RViz | Проверить Fixed Frame = `map` |
| Hector SLAM не найден | Выполнить `source /opt/ros/noetic/setup.bash` |
| Карта пустая (вся серая) | Физически перемещайте LiDAR! |
| "Failed to contact master" | Не запущен roscore (Терминал 1) |

## 🎯 Ключевые выводы

1. **Карта обновляется только при движении LiDAR** — нужно ходить с датчиком!
